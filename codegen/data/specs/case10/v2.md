# topo_sort

Given a directed graph on vertices `0..n-1` and a list of directed `edges` (`[u, v]` means an edge
`u -> v`, i.e. "u must come before v"), return the **lexicographically smallest** topological
ordering of all `n` vertices as a list of ints. If the graph has a cycle (no topological ordering
exists), return `[]`.

Pinned rules:

- "Lexicographically smallest" = Kahn's algorithm with min-selection: at each step, among all
  vertices that currently have no unmet predecessors, choose the **smallest-numbered** one. This
  yields a unique, deterministic order.
- An acyclic result contains **all** `n` vertices, each exactly once, in dependency order.
- `n` is an integer `>= 0`. `n == 0` returns `[]`.
- **Duplicate edges** are allowed and treated as a single edge — in-degree must be counted with
  duplicates de-duplicated (a repeated `[u, v]` contributes to `v`'s in-degree only once).
- A **self-loop** (`[u, u]`) is a cycle → return `[]`.
- **Raise `ValueError`** if any edge endpoint is out of range (`< 0` or `>= n`), if `n < 0`, or if an
  edge is not a 2-element list.
- Return a list of ints.

## Examples

- `topo_sort(2, [[1, 0]])` → `[1, 0]`
- `topo_sort(2, [[0, 1], [1, 0]])` → `[]`
- `topo_sort(4, [[0, 1], [0, 1], [1, 2]])` → `[0, 1, 2, 3]`
- `topo_sort(3, [[0, 5]])` raises `ValueError`
